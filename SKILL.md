# Stop Slop (Долой шаблоны)

Устраните предсказуемые ИИ-паттерны из текста.

## Основные правила

1. **Вырезайте слова-паразиты.** Убирайте вступительные фразы-«расчистки горла», лексические костыли для ложного акцента и все наречия. См. [references/phrases.md](references/phrases.md).
2. **Избегайте шаблонных конструкций.** Не используйте бинарные противопоставления, перечисления через отрицание, драматическую фрагментацию, риторические зачины, ложную олицетворённость. См. [references/structures.md](references/structures.md).
3. **Будьте конкретны.** Никаких размытых утверждений («Причины носят структурный характер»). Называйте конкретную вещь. Никаких ленивых крайностей («всегда», «никогда», «каждый»), делающих размытую работу.
4. **Документируйте всё. Ничего не подразумевайте**
5. **Будьте воспроизводимы.** Описывайте действия и результаты так, чтобы их мог повторить другой человек. Не «иногда падает», а «при 3-м запросе за 1 секунду».

## Быстрая проверка

Прежде чем отдать текст:

* Есть наречия? Убейте их.
* Есть расчистка горла в духе «вот что/это/то самое»? Рубите до сути.
* Есть противопоставление «не X, а Y»? Сформулируйте сразу Y.
* Есть неопределённая декларация («Последствия значительны»)? Назовите конкретное следствие.
* Есть мета-связки («В этом разделе мы рассмотрим»)? Удалите. Пусть документ сам говорит за себя.
* Есть «иногда», «примерно», «в большинстве случаев»? Замените на конкретные условия.
* Описан ожидаемый результат? Не «должно работать», а конкретно (код, состояние, значение).
* Действия воспроизводимы? Указаны входные данные, окружение, версия (если применимо).

## Оценка

Оцените по шкале 1–10 по каждому параметру:

| Параметр | Вопрос |
|----------|--------|
| Прямота | Утверждения или объявления? |
| Конкретность | Названы точные значения, условия, сущности? |
| Доверие | Текст написан для людей, которые понимают контекст? |
| Плотность | Есть ли что-то, что можно сократить без потери смысла? |

Если сумма ниже 35/40 — требуется доработка.
